# Документация: как поддерживать

## Правило
Изменил API или поведение backend -> в этом же PR обнови `docs/API_V1.md`.

## Минимальный процесс
1. Перед задачей: проверь, затрагивает ли она контракт API.
2. Если да: обнови `docs/API_V1.md` (request/response/errors/enums).
3. Запусти `./gradlew test`.
4. В описании PR добавь секцию `API changes`:
   - что изменилось
   - обратная совместимость (да/нет)
   - что нужно поменять в Android-клиенте

## Версионирование API
- Пока используйте `v1` (этот файл).
- Ломающие изменения:
  - либо новый endpoint/поле без удаления старого,
  - либо `v2` документ и постепенная миграция.

## Definition of Done для backend-задачи
- Код реализован
- Тесты зелёные
- `docs/API_V1.md` актуален
- Логи не содержат секретов
